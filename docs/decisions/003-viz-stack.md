# Decision 003: Visualization Stack

**Date:** 2025-04

## Context

We need an interactive visualization that deploys as a static site with no server,
loads quickly, and is maintainable by someone who isn't a front-end specialist.

## Decision

**Observable Plot** loaded from CDN, plain ES modules, no build step.

- HTML + CSS + vanilla JS modules
- Observable Plot for charts (CDN: `cdn.jsdelivr.net/npm/@observablehq/plot`)
- Pre-aggregated JSON data files (generated by pipeline step 05)
- GitHub Pages for hosting

## Alternatives considered

| Option | Why not used |
|---|---|
| D3.js directly | More verbose; Observable Plot is a higher-level wrapper built on D3 |
| React + Vite | Build step required; overkill for a static analysis site |
| Svelte / SvelteKit | Same issue; unnecessary complexity |
| Observable notebooks | Not easily self-hosted; harder to integrate with pipeline output |
| Plotly | Heavier bundle; less composable |
| Recharts / Chart.js | React/DOM coupling; less flexible for custom layouts |

## Why Observable Plot

- Concise, expressive grammar-of-graphics API
- Excellent support for area bands (IQR), regression lines, tooltips
- No build step required — loads as an ES module from CDN
- Strong defaults for dark-themed charts
- Actively maintained by Observable (the team behind D3)

## Why pre-aggregated JSON

The browser does zero aggregation. All cohort statistics (percentiles, means, totals)
are computed in the Python pipeline and written as compact JSON. This means:
- Fast load times (files are small, ~1-5 MB total for 10k users)
- No risk of browser OOM on large datasets
- The viz works identically in local dev and deployed environments

## Consequences

- No framework = no hot reload; refresh manually during development
- Pre-aggregation means some interactive drill-down requires re-running the pipeline
- The `viz/data/` directory must be kept in sync with `data/viz/` (done by `scripts/dev.sh` and the deploy workflow)
- Adding a new chart requires writing a new JS module in `viz/charts/`
